Necesito implementar Forgot Password 
con reset via email en Slably.

PASO 1 — DB Schema
En lib/db/src/schema/ crea archivo: 
passwordResets.ts

export const passwordResets = pgTable(
  'password_resets', {
  id: serial('id').primaryKey(),
  userId: integer('user_id')
    .notNull().references(() => users.id),
  token: text('token').notNull().unique(),
  expiresAt: timestamp('expires_at')
    .notNull(),
  usedAt: timestamp('used_at'),
  createdAt: timestamp('created_at')
    .defaultNow().notNull(),
});

Exporta desde lib/db/src/schema/index.ts
Haz db push.

PASO 2 — Backend
En artifacts/api-server/src/routes/auth.ts
agrega 2 endpoints:

POST /api/auth/forgot-password
  Body: { email: string }
  
  1. Busca el usuario por email
  2. Si no existe → responde 200 
     (no revelar si existe o no)
  3. Genera token: 
     crypto.randomBytes(32).toString('hex')
  4. Guarda en password_resets:
     { userId, token, 
       expiresAt: now + 1 hora }
  5. Envía email con Resend desde 
     hello@slably.app:
     
     Subject: "Reset your Slably password"
     Body HTML:
     <div style="font-family: Arial; 
       max-width: 500px; margin: 0 auto;">
       <img src="https://slably.app/slably-logo.png" 
         height="40" />
       <h2>Reset your password</h2>
       <p>You requested a password reset 
       for your Slably account.</p>
       <a href="${APP_URL}/reset-password?token=${token}"
         style="background: #F97316; 
         color: white; padding: 12px 24px; 
         border-radius: 8px; 
         text-decoration: none;
         display: inline-block; 
         margin: 16px 0;">
         Reset Password
       </a>
       <p style="color: #666; font-size: 14px;">
         This link expires in 1 hour. 
         If you didn't request this, 
         ignore this email.
       </p>
     </div>
     
  6. Responde: { message: "If an account 
     exists with that email, you will 
     receive a reset link." }

POST /api/auth/reset-password
  Body: { token: string, 
          newPassword: string }
  
  1. Busca el token en password_resets
  2. Verifica que no esté usado 
     (usedAt IS NULL)
  3. Verifica que no esté expirado
     (expiresAt > now)
  4. Si inválido → 400 "Invalid or 
     expired reset token"
  5. Hashea newPassword con bcrypt
  6. Actualiza users.password_hash
  7. Marca token como usado:
     UPDATE password_resets 
     SET usedAt = now()
  8. Responde: { message: 
     "Password updated successfully" }

3C. Crea src/pages/auth/ResetPassword.tsx
Lee el token de la URL: 
  ?token=xxx (useLocation + URLSearchParams)

Si no hay token → redirige a /login

Formulario:
  Título: "Create new password"
  
  New Password * 
    (min 8 chars, show/hide toggle)
  Confirm Password *
    (debe coincidir)
  
  Botón naranja: "Update Password"
  → POST /api/auth/reset-password
    con { token, newPassword }
  
Estado de éxito:
  ✅ "Password updated successfully!"
  Botón: "Sign In" → navega a /login

Estado de error (token inválido):
  ❌ "This reset link is invalid 
  or has expired."
  Botón: "Request new link" 
  → navega a /forgot-password
