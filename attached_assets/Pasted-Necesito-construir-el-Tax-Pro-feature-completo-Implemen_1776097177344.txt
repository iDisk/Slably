PASO 2 — Backend Routes
Crea artifacts/api-server/src/routes/taxPro.ts
con estos endpoints:

GET /api/tax-pro/clients
  → Lista todos los builders conectados 
    al tax pro autenticado
  → Para cada builder devuelve:
    { id, name, email, companyName,
      projectCount, totalInvoiced,
      totalExpenses, totalCollected,
      pendingW9Count }

GET /api/tax-pro/clients/:builderId/summary
  → Resumen financiero del builder
    para el año fiscal (query param: year)
  → Devuelve:
    { totalInvoiced, totalCollected,
      totalExpenses, estimatedProfit,
      marginPercent,
      expensesByCategory: {
        materials, labor, equipment,
        permits, other
      },
      projectCount, invoiceCount,
      expenseCount }

GET /api/tax-pro/clients/:builderId/expenses
  → Lista paginada de gastos
  → Filtros: year, category, projectId
  → Incluye: amount, vendor, category,
    date, projectName, receiptUrl

GET /api/tax-pro/clients/:builderId/invoices
  → Lista paginada de invoices
  → Filtros: year, status
  → Incluye: invoiceNumber, title,
    total, status, paidAt, projectName

GET /api/tax-pro/clients/:builderId/vendors
  → Lista de vendors/subcontratistas
    con totales pagados en el año
  → Incluye: name, totalPaid, w9Status,
    needs1099 (totalPaid > 600)

POST /api/tax-pro/invite/:builderEmail
  → Tax pro invita a un builder por email
  → Crea registro en tax_pro_clients 
    con status='pending'
  → Envía email al builder con link 
    para aceptar

POST /api/tax-pro/accept/:taxProId
  → Builder acepta la invitación
  → Actualiza status='active' y acceptedAt

POST /api/tax-pro/vendors/:vendorId/request-w9
  → Envía email al vendor solicitando W-9
  → Email desde hello@slably.app:
    "Subject: W-9 Request from [Builder Name]"
    Body: "[Builder] is requesting your W-9 
    for tax purposes. Please reply to this 
    email with your completed W-9 form."

Registra el router en 
artifacts/api-server/src/routes/index.ts:
import taxProRouter from "./taxPro";
