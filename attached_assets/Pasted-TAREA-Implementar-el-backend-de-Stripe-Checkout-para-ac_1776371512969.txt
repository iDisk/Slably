PIEZA 2 — Endpoint de Checkout
Crea: artifacts/api-server/src/routes/billing.ts

POST /api/billing/checkout
- requireAuth — solo builders autenticados
- Body: { plan: 'starter' | 'pro' | 'business' }
- Valida con Zod que plan sea uno de los tres valores válidos
- Lógica:
  1. Obtiene la organización del usuario desde DB
  2. Si org.stripeCustomerId existe, lo usa
     Si no existe, crea un Customer en Stripe con email y name del usuario,
     guarda el ID en DB (UPDATE organizations SET stripe_customer_id)
  3. Crea una Stripe Checkout Session con:
     - mode: 'subscription'
     - customer: stripeCustomerId
     - line_items: [{ price: PLAN_PRICE_IDS[plan], quantity: 1 }]
     - success_url: process.env.APP_URL + '/billing/success?session_id={CHECKOUT_SESSION_ID}'
     - cancel_url: process.env.APP_URL + '/pricing'
     - metadata: { organizationId, plan }
     - subscription_data.trial_period_days: 30 (solo si plan === 'starter')
  4. Devuelve: { url: checkoutSession.url }
- Error si PLAN_PRICE_IDS[plan] está vacío: 
  500 con { error: 'stripe_not_configured', message: 'Stripe price IDs not configured' }

PIEZA 3 — Webhook de Stripe
Crea: artifacts/api-server/src/routes/stripe-webhook.ts

POST /api/webhooks/stripe
- SIN requireAuth — es llamado por Stripe directamente
- CRÍTICO: necesita el body RAW (Buffer), no parseado como JSON
  Usa express.raw({ type: 'application/json' }) como middleware de esta ruta
  ANTES de que el body parser global lo intercepte
- Verifica la firma con stripe.webhooks.constructEvent(
    req.body, req.headers['stripe-signature'], STRIPE_WEBHOOK_SECRET
  )
- Si la firma falla: 400 con { error: 'invalid_signature' }

Maneja estos 4 eventos:

1. checkout.session.completed
   - session.metadata.organizationId + session.metadata.plan
   - UPDATE organizations SET 
       subscription_plan = plan,
       stripe_subscription_id = session.subscription,
       stripe_subscription_status = 'active'
     WHERE id = organizationId

2. customer.subscription.updated
   - Busca la org por stripe_subscription_id = subscription.id
   - Mapea subscription.items.data[0].price.id → plan name usando PLAN_PRICE_IDS
   - UPDATE subscription_plan + stripe_subscription_status = subscription.status

3. customer.subscription.deleted
   - Busca la org por stripe_subscription_id
   - UPDATE subscription_plan = 'free', stripe_subscription_status = 'canceled'

4. invoice.payment_failed
   - Busca la org por stripe_customer_id = invoice.customer
   - UPDATE stripe_subscription_status = 'past_due'

Cualquier otro evento: responde 200 sin hacer nada (no falles en eventos desconocidos).
